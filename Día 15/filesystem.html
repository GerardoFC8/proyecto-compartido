Entendiendo el Filesystem en Laravel

Laravel proporciona una potente abstracción del sistema de archivos gracias al paquete Flysystem de Frank de Jonge. Esto te permite interactuar con diferentes sistemas de almacenamiento (tu disco local, Amazon S3, etc.) usando la misma API unificada.

Toda la configuración se centraliza en el archivo config/filesystems.php. Esta guía te explicará las partes clave de este archivo y cómo funcionan.

¿Por qué es tan útil?

Puedes desarrollar tu aplicación usando el disco local y, cuando pases a producción, simplemente cambiar una variable de entorno para empezar a usar S3, sin cambiar ni una línea de tu código de lógica de archivos.

Disco por Defecto

Esta es la primera configuración importante. Define qué "disco" se usará por defecto cuando no especifiques uno explícitamente.


'default' => env('FILESYSTEM_DISK', 'local'),
  • 'default' => 'local': Por defecto, Laravel usará el disco llamado 'local'.
  • env('FILESYSTEM_DISK', ...): Es una excelente práctica controlar esto con una variable de entorno. En tu archivo .env, puedes cambiar FILESYSTEM_DISK=local a FILESYSTEM_DISK=s3 para cambiar el almacenamiento de toda la aplicación.

El Disco 'local' (Privado)

Este disco está configurado para almacenar archivos que no deben ser accesibles públicamente a través de una URL. Es ideal para logs, archivos de sistema o cualquier archivo que un usuario suba y que solo deba ser accesible a través de tu aplicación (por ejemplo, una factura que solo el usuario autenticado puede descargar).


'local' => [
    'driver' => 'local',
    'root' => storage_path('app/private'),
    'throw' => false,
],
  • 'driver' => 'local': Le dice a Laravel que use el sistema de archivos del servidor.
  • 'root' => storage_path('app/private'): Especifica que todos los archivos de este disco se guardarán dentro del directorio storage/app/private. Este directorio, por defecto, no es accesible desde la web, lo que lo hace seguro.

El Disco 'public' (Público)

Este es el disco que debes usar para almacenar archivos que sí necesitan ser accesibles desde el navegador, como avatares de usuario, imágenes de posts, CSS o JS compilados, etc.


'public' => [
    'driver' => 'local',
    'root' => storage_path('app/public'),
    'url' => env('APP_URL').'/storage',
    'visibility' => 'public',
    'throw' => false,
],
  • 'root' => storage_path('app/public'): Los archivos se guardan en storage/app/public.
  • 'url' => env('APP_URL').'/storage': Aquí está la magia. Define la URL base para acceder a estos archivos. Si tu APP_URL es http://localhost, un archivo en storage/app/public/avatars/user.jpg será accesible en http://localhost/storage/avatars/user.jpg.
  • 'visibility' => 'public': Asegura que los archivos tengan los permisos correctos para ser visibles públicamente.
¡Importante! Para que el disco public funcione, necesitas crear un enlace simbólico. Ver la sección correspondiente.

El Disco 's3' (Cloud Storage)

Laravel viene pre-configurado para usar Amazon S3, un servicio de almacenamiento en la nube muy popular. Es perfecto para aplicaciones escalables, ya que desacopla el almacenamiento de archivos de tu servidor web.


's3' => [
    'driver' => 's3',
    'key' => env('AWS_ACCESS_KEY_ID'),
    'secret' => env('AWS_SECRET_ACCESS_KEY'),
    'region' => env('AWS_DEFAULT_REGION'),
    'bucket' => env('AWS_BUCKET'),
    'url' => env('AWS_URL'),
    // ... otras opciones
],

Para usarlo, necesitas instalar el driver de S3:

composer require --with-all-dependencies league/flysystem-aws-s3-v3 "^3.0"

Luego, simplemente llena las credenciales de AWS en tu archivo .env. Puedes tener discos "públicos" y "privados" en S3 también, controlando la visibilidad de los archivos al subirlos.

Enlaces Simbólicos

Como vimos, los archivos del disco public se guardan en storage/app/public, pero el directorio raíz público de tu servidor web es la carpeta public/. ¿Cómo se conectan?

La respuesta es un enlace simbólico (symlink). Laravel lo facilita con un simple comando de Artisan.


'links' => [
    public_path('storage') => storage_path('app/public'),
],

Esta configuración le dice al comando php artisan storage:link que cree un enlace llamado storage dentro de tu carpeta public/ que apunte directamente a storage/app/public/.

Ejecuta este comando:

php artisan storage:link

Solo necesitas ejecutar este comando una vez. Ahora, tu servidor web puede acceder a los archivos en storage/app/public como si estuvieran en public/storage.

Helpers de Rutas: asset() vs. storage_path()

Esta es una de las confusiones más comunes. La clave es entender **quién** necesita la ruta: ¿el navegador del usuario o tu servidor PHP?

La Analogía Clave

Imagina que tu servidor es tu casa y el navegador es un turista.

  • El Servidor (PHP) conoce la dirección exacta de cada objeto en tu casa (la ruta del sistema de archivos): C:\Users\Tu\Proyecto\storage\....
  • El Navegador (Turista) solo conoce las direcciones públicas que le das para visitar (URLs): http://tu-sitio.com/imagenes/....

Nunca le darías a un turista la dirección interna de tu casa, ¡no sabría cómo llegar!

Cuándo usar cada uno

Helper Lo que genera Contexto de Uso Ejemplo
asset() Una URL pública completa Para el Navegador. Siempre en vistas Blade dentro de atributos HTML como <img src="">, <link href="">, etc. <img src="{{ asset('storage/imagen.jpg') }}">
storage_path() Una ruta absoluta del servidor Para el Servidor (PHP). Para operaciones de archivos, verificar si existen (file_exists) o para librerías como DomPDF. $pdf = Pdf::loadView('vista', $data);
$path = storage_path('app/private/doc.pdf');
public_path() Una ruta absoluta del servidor a la carpeta public Para el Servidor (PHP). Menos común, pero útil si necesitas interactuar con archivos directamente en tu carpeta public. $path = public_path('css/style.css');
Regla de Oro: Si la ruta va a ser usada por el navegador para pedir un archivo, usa asset(). Si tu código PHP necesita leer, escribir o encontrar un archivo en el disco del servidor, usa storage_path() o public_path().

Uso Básico con el Facade `Storage`

Una vez configurados tus discos, interactuar con ellos es increíblemente fácil usando el facade Storage.

Guardar contenido en un archivo


use Illuminate\Support\Facades\Storage;

// Usando el disco por defecto (ej. 'local')
Storage::put('documentos/archivo.txt', 'Contenido del archivo');

// Especificando un disco (ej. 'public')
Storage::disk('public')->put('avatars/1/avatar.jpg', $contenidoDelArchivo);

Manejo de Subida de Archivos (Request Files)

Cuando trabajas con archivos subidos desde un formulario (instancias de Illuminate\Http\UploadedFile), Laravel ofrece métodos muy convenientes.


// En tu controlador
public function store(Request $request)
{
    if ($request->hasFile('gallery_image')) {
        $file = $request->file('gallery_image');
        $fileName = time() . '_' . $file->getClientOriginalName();

        // Opción 1: Método directo sobre el objeto UploadedFile
        // El tercer parámetro 'public' es el nombre del disco a usar.
        $path1 = $file->storeAs('posts/gallery', $fileName, 'public');

        // Opción 2: Usando el Facade Storage (resultado idéntico)
        $path2 = Storage::disk('public')->putFileAs('posts/gallery', $file, $fileName);

        // Ambas variables, $path1 y $path2, contendrán la misma ruta:
        // 'posts/gallery/1678886400_nombre_imagen.jpg'
    }
}
¿Cuál es la diferencia?

Funcionalmente, no hay ninguna diferencia real entre $file->storeAs('posts/gallery', $fileName, 'public') y Storage::disk('public')->putFileAs('posts/gallery', $file, $fileName).

El método storeAs() en el objeto UploadedFile es simplemente un atajo conveniente que llama internamente al facade Storage. La elección entre uno y otro es principalmente una cuestión de preferencia y estilo de código. Muchos desarrolladores prefieren usar los métodos directos del archivo ($file->storeAs()) por ser más fluidos y directos en el contexto de un request.

Obtener la URL de un archivo público


// Devuelve: /storage/avatars/1/avatar.jpg (si usas el driver local)
// O la URL completa de S3 si usas ese driver.
$url = Storage::disk('public')->url('avatars/1/avatar.jpg');

// Usando el helper asset() para una vista Blade:
// Genera: http://tu-dominio.com/storage/avatars/1/avatar.jpg
echo asset(Storage::disk('public')->url('avatars/1/avatar.jpg'));

Verificar si un archivo existe


if (Storage::disk('local')->exists('documentos/archivo.txt')) {
    // El archivo existe
}

Descargar un archivo (desde un controlador)


// Para archivos en el disco 'local' (privado)
return Storage::download('facturas/factura-001.pdf');

// Puedes darle un nombre diferente al descargar
return Storage::download('facturas/factura-001.pdf', 'Mi Factura.pdf');
